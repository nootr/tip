TIP GitHub Spec

Trust Infrastructure Protocol

Portable trust, not another login.

TIP is an open protocol for signed claims and attestations that can travel between apps, communities, platforms, and AI agents.

View on GitHub Read the draft spec

Cryptographic identities

TIP identities are Ed25519 public keys. Handles like GitHub usernames or domains are verifiable claims, not the root identity.

Signed events

Claims, attestations, updates, and revocations are append-only signed events. Nodes store evidence; clients verify it.

Local trust policies

There is no global reputation score. Each app decides which issuers and signals matter for its own context.

Developer trust demo

tip key generate

tip identity create --out identity.json

tip claim add github joris \
  --proof-url https://gist.github.com/... \
  --out github-claim.json

tip claim revoke <claim-event-id>

tip attest issue <subject-pubkey> trusted_contributor \
  --message "Useful open-source contributor"

tip attest revoke <subject-pubkey> <attestation-event-id>
