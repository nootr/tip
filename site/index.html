Trust Infrastructure Protocol · TIP 0.1 alpha

Portable trust evidence for an internet of agents.

TIP is an open-source protocol for signed identities, claims, attestations, and revocations. No blockchain. No global reputation score. Just verifiable public evidence that can move between systems.

Protocol model

Trust is evidence, not a score.

TIP keeps the base layer small: keys sign events, nodes store valid public evidence, and clients choose their own trust policy.

01

Cryptographic identity

The canonical identity is an Ed25519 public key. Handles, domains, and usernames are claims.

02

Signed event log

Claims, attestations, and revocations are append-only signed events with stable SHA-256 IDs.

03

Local trust policy

Every app decides which issuers, claims, and attestations matter in its own context.

How it moves

From key to portable evidence.

  1. Generate key

    Create a local Ed25519 identity.

  2. Publish claims

    Sign handles, links, or public statements.

  3. Receive attestations

    Other issuers sign statements about your key.

  4. Sync nodes

    Explicit peers exchange valid events with cursor pagination.

Reference node

Run a small HTTP node with peer sync.

The Rust node stores events in SQLite, exposes a compact HTTP API, and can periodically pull from explicitly configured peers. Run one as a community registry, public evidence mirror, or local trust-policy cache.

Node config

[node]
bind = "127.0.0.1:8080"
db = "tip-node.sqlite3"
key = "tip-node-key.json"
name = "Local TIP Node"
description = "Community trust registry"
website = "https://example.com"
contact = "mailto:admin@example.com"

[sync]
on_start = true
limit = 500
periodic_seconds = 300
full_resync_seconds = 86400

[[peers.nodes]]
url = "http://127.0.0.1:8081"
expected_node_public_key = "peer-node-public-key"
name = "Community peer"

Quick start

Create, submit, and query trust events.

cargo install --path crates/tip-cli
cargo install --path crates/tip-node

cp tip-node.example.toml tip-node.toml
tip-node serve --config tip-node.toml

tip key generate --name default
tip identity create --out identity.json
tip claim add github joris --out github-claim.json

tip event submit-batch identity.json github-claim.json \
  --node http://127.0.0.1:8080

tip query --subject <public-key> --limit 100 \
  --node http://127.0.0.1:8080

Build with it

TIP is early, useful, and intentionally small.

Start with the spec, run a node, submit events, and decide what trust means in your application.
